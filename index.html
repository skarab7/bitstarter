Currently, there is no simple REPL tool for Infrastructure-as-Code solutions. A tool that would let you to understand instantly changes in your infrastructure after running your code. We believe that the text log is not enough.

In our project, we want to fill this gap with Serpent. We build a REPL tool for chef. The tool will visualize how provisioned machines changes after applying your code. Serpent provides near-instant feedback. With the rewind functionality, you can step-back and more-forward in your code and you see immediately changing state of your machine. Similar to other REPL tools, you can apply fragments of your code.

We will implement the tool as an add-on for Sublime2. The tool will be open-sourced (GPL). The first version will support chef. Initially, we will support a single-machine provisioning, next we will focus on supporting you in developing mutli-node environments.

Later, we will work on plugins for other editors and configuration management platforms.

FAQ

Do you plan to extend Serpent to other configuration management platforms?
Yes. We start with Chef, because we have been working with this technology in for last years. It is a big challenge to make the Serpent fast and responsive. So, we prefer to work with well-known technology. We will support more platforms in future.
How the tool will evaluate the chef code?
The Serpent will run in background multiple VMs. The VMs can be LXC containers or virtual machines in AWS or Openstack. The Serpent will run one or more VMs to make the tool react fast on any code changes. Each VM will contain different state of the provisioned machine.
What is the big picture?
The Serpent is an essential piece for new type of environment building tool. The tool will deliver an interactive way of building your infrastructure with near-instant feedback.
